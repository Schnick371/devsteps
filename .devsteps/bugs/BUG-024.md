# Bug: Story Items Cannot Be Opened in Editor

## Problem

When clicking a story item in TreeView or using "Reveal in Explorer", the extension tries to open:
```
.devsteps/storys/STORY-001.md  ❌
```

But the actual file is at:
```
.devsteps/stories/STORY-001.md  ✅
```

**Error:** `Unable to resolve nonexistent file 'storys/STORY-001.md'`

## Root Cause

Two commands use naive pluralization (`${item.type}s`) instead of `TYPE_TO_DIRECTORY` mapping:

**Line 325 (`devsteps.openItem`):**
```typescript
const itemTypeFolder = `${item.type}s`; // story → storys ❌
```

**Line 641 (`devsteps.revealInExplorer`):**
```typescript
const itemTypeFolder = `${item.type}s`; // story → storys ❌
```

## Why It Breaks

English pluralization isn't always "+s":
- ✅ `task → tasks`, `epic → epics`, `bug → bugs` (work by accident)
- ❌ `story → storys` (should be `stories`)

## Solution

Import `TYPE_TO_DIRECTORY` from `@schnick371/devsteps-shared`:
```typescript
import { addItem, getItem, updateItem, listItems, TYPE_TO_DIRECTORY } from '@schnick371/devsteps-shared';
```

Replace both occurrences:
```typescript
const itemTypeFolder = TYPE_TO_DIRECTORY[item.type]; // story → stories ✅
```
