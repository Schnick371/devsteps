# Bug: Welcome View bleibt nach MCP Init sichtbar

## Problem
Nach MCP-basierter Projektinitialisierung via `@devsteps #mcp_devsteps_devsteps-init` bleibt Welcome View sichtbar, obwohl `.devsteps/` existiert.

**Aktuelles Verhalten:**
1. User nutzt Copilot Chat: `@devsteps #mcp_devsteps_devsteps-init MyProject`
2. MCP Server erstellt `.devsteps/` Directory erfolgreich
3. FileSystemWatcher erkennt creation und bietet Reload an
4. **User MUSS manuell "Reload Window" Button klicken**
5. Welcome View bleibt bis reload sichtbar

**Root Cause:**
```typescript
// extension.ts:122-133
devstepsWatcher.onDidCreate(async () => {
  // Context keys werden upgedatet ✓
  await vscode.commands.executeCommand('setContext', 'devsteps.showWelcome', false);
  
  // PROBLEM: Wartet auf User-Input statt automatic reload
  const reload = await vscode.window.showInformationMessage(
    'DevSteps project initialized! Reload window to activate extension.',
    'Reload Window'
  );
  if (reload) {  // ← User muss klicken!
    await vscode.commands.executeCommand('workbench.action.reloadWindow');
  }
});
```

## Expected Behavior
Nach MCP init sollte Extension **automatisch** reloaden ohne User-Interaction.

## User Impact
- **Verwirrend**: Welcome View zeigt "No Project" obwohl init erfolgreich
- **Extra Click**: User muss manuell reload button finden und klicken
- **Inkonsistent**: CLI init via Terminal triggert keinen automatic reload

## Reproduction
1. Open empty workspace
2. Extension shows Welcome View
3. Use Copilot Chat: `@devsteps #mcp_devsteps_devsteps-init TestProject`
4. Observe: Welcome View bleibt, Info Message erscheint
5. MUST CLICK: "Reload Window" Button

## Solution
**Automatic Reload** nach `.devsteps` creation:
```typescript
devstepsWatcher.onDidCreate(async () => {
  logger.info('.devsteps directory created - reloading extension');
  await vscode.commands.executeCommand('setContext', 'devsteps.showWelcome', false);
  await vscode.commands.executeCommand('setContext', 'devsteps.hasProject', true);
  
  // Show notification WITHOUT awaiting user choice
  vscode.window.showInformationMessage(
    'DevSteps project initialized! Reloading window...'
  );
  
  // Immediate reload - no user interaction needed
  await vscode.commands.executeCommand('workbench.action.reloadWindow');
});
```

## Acceptance Criteria
- [ ] After MCP init, window reloads automatically
- [ ] No manual "Reload Window" button click needed
- [ ] Works for both Copilot Chat and CLI init paths
- [ ] User sees notification explaining reload
- [ ] After reload, TreeView shows (no Welcome View)

## Testing
1. Empty workspace → Welcome View visible
2. Copilot: `@devsteps #mcp_devsteps_devsteps-init Test`
3. Verify: Window reloads automatically within 1 second
4. Verify: After reload, TreeView visible, Welcome View gone
5. Repeat with CLI init via terminal
