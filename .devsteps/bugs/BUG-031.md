# TreeView Duplicate IDs Bug

## Problem
After TreeView refresh, children on levels 2+3 disappear with error:
```
Element with id EPIC-008 is already registered
```

## Reproduction Steps
1. Open DevSteps TreeView in Hierarchical mode
2. Expand Epic ‚Üí Children visible ‚úÖ
3. Refresh TreeView (F5 or file change)
4. Expand Epic ‚Üí Children missing ‚ùå

## Root Cause
`WorkItemNode.toTreeItem()` Line 48:
```typescript
treeItem.id = this.item.id;  // ‚Üê NOT UNIQUE!
```

VS Code TreeView requires **globally unique** `treeItem.id` values.
Same item (e.g., EPIC-008) can appear multiple times:
- As root node in "üå≤ Scrum Hierarchy"
- As child via relates-to relationship
- As blocked-by dependency

## Impact
- **Critical:** TreeView hierarchy broken after refresh
- **User Experience:** Confusing - data appears/disappears
- **Workaround:** Reload VS Code window (temporary)

## Affected Code
- `packages/extension/src/treeView/nodes/workItemNode.ts` Line 48
- All hierarchical relationships (implemented-by, relates-to, blocks, etc.)

## Solution Approach
Generate unique IDs by combining:
1. Item ID (EPIC-008)
2. Parent context (hierarchy root or parent item ID)
3. Relationship type (implemented-by, relates-to, etc.)

Example: `hierarchy-scrum-EPIC-008` vs `STORY-001-relates-to-EPIC-008`
