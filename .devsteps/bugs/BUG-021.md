# Hide Done Button Enablement Bug

## Problem
Der "Hide Done Items" Button wird durch `"enablement": "false"` deaktiviert wenn er aktiv ist. Das ist falsche UX - Toggle Buttons sollen **immer** clickable bleiben.

**Aktuelles Verhalten:**
- Button ist aktiv (Hide Done ist an) → Button wird disabled (`enablement: false`)
- User kann nicht mehr zurück zu "Show Done" wechseln
- Inkonsistente UX - andere Toggle buttons bleiben enabled

**Erwartetes Verhalten:**
- Button ist immer clickable/enabled 
- Zeigt current state durch Checkmark (✓), nicht durch disabled state
- User kann immer between den beiden states togglen

## Root Cause
**File:** `packages/extension/package.json` Line 243
```json
{
  "command": "devsteps.hideDone.active",
  "title": "$(check) Hide Completed Items", 
  "enablement": "false"  // ← WRONG! Toggle should never be disabled
}
```

## VS Code Best Practice
Toggle buttons should use visual indicators (checkmarks) but remain enabled:
- ✅ Use `toggled` property for checkmarks
- ✅ Remove `enablement: false` - both variants should be clickable
- ✅ Show state through visual feedback, not disabled state

## User Impact
- **Critical UX Bug**: User cannot toggle back to "show done" 
- **Inconsistent**: Other toggles (viewMode, hierarchy) work correctly
- **Confusing**: Disabled button suggests broken functionality

## Solution
Remove `"enablement": "false"` from active command definition - both active and inactive commands should be clickable.
