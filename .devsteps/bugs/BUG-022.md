# Command Name Typos - Missing Dots Breaking Core Functionality

## Problem
Multiple `executeCommand` calls have typos in command names, missing the dot separator. This causes "command not found" errors for core functionality.

**Broken Commands:**
1. `devstepsopenItem` → Should be `devsteps.openItem`
2. `devstepsupdateStatus` → Should be `devsteps.updateStatus`

**User Impact:**
- ❌ Search functionality fails with "command 'devstepsopenItem' not found"
- ❌ Dashboard "Open Item" buttons don't work
- ❌ Status update operations fail
- ❌ Create item "Open" action broken

## Root Cause Analysis
VS Code command naming convention requires dot notation: `extension.command`.
All commands are correctly registered in package.json as:
- `devsteps.openItem`
- `devsteps.updateStatus`

But `executeCommand` calls have typos missing the dots:
- `devstepsopenItem` (3 occurrences)
- `devstepsupdateStatus` (1 occurrence)

## Affected Files & Lines

### packages/extension/src/commands/index.ts
- **Line 280**: `devstepsopenItem` → `devsteps.openItem`
- **Line 504**: `devstepsopenItem` → `devsteps.openItem`
- **Line 742**: `devstepsupdateStatus` → `devsteps.updateStatus`

### packages/extension/src/webview/dashboardPanel.ts
- **Line 212**: `devstepsopenItem` → `devsteps.openItem`

## VS Code Best Practice (per documentation)
Command IDs must follow naming convention:
- ✅ `extensionName.commandName` (with dot)
- ❌ `extensionnamecommandname` (no dot)

## Error Evidence
User reported: "Error searching items: command 'devstepsopenItem' not found"

## Solution
Simple string fixes - add missing dots in 4 locations.

- Dashboard item buttons → Should work
