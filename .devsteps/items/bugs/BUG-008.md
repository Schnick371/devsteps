# Keybinding Conflict: Priority Filter vs Command Palette

## Problem
`devsteps.filterByPriority` uses **Ctrl+Shift+P** which conflicts with VS Code's Command Palette - the most important shortcut in VS Code!

**Current (package.json line 231):**
```json
{
  "command": "devsteps.filterByPriority",
  "key": "ctrl+shift+p",
  "mac": "cmd+shift+p",
  "when": "focusedView == devsteps.itemsView"
}
```

Even with `when` clause, this creates confusion and muscle memory conflicts.

## Research Findings
VS Code best practices:
- Use **chord shortcuts** for extension commands (two-key sequences)
- Avoid conflicts with core VS Code shortcuts
- Examples: `Ctrl+K Ctrl+S`, `Ctrl+Shift+D D`

## Proposed Solutions

### Option A: Chord with K (Recommended)
```json
"key": "ctrl+k p",
"mac": "cmd+k p"
```
- **Pros**: Short, follows VS Code pattern (`Ctrl+K` prefix common)
- **Cons**: None

### Option B: DevSteps namespace chord
```json
"key": "ctrl+shift+r p",
"mac": "cmd+shift+r p"  
```
- **Pros**: Clear DevSteps namespace (R = devsteps)
- **Cons**: Longer sequence

### Option C: Single alternative key
```json
"key": "ctrl+shift+i",
"mac": "cmd+shift+i"
```
- **Pros**: Single keystroke
- **Cons**: Might conflict with other extensions

## Recommendation
**Use Option A: `Ctrl+K P`**
- Follows VS Code conventions
- No conflicts
- Easy to remember (K for "keyboard command", P for "priority")
- Consistent with other VS Code chord shortcuts
