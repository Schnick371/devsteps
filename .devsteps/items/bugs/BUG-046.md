# Problem

The extension's TreeView shows incorrect priority information:

1. **Icon colors don't match priority** - Icons don't get colored according to priority levels
2. **Tooltip shows wrong priority** - Hovers display incorrect/undefined priority values

## Observed Behavior

- TreeView items show icons without priority-based coloring
- Hovering over items shows `Priority: undefined` or Eisenhower quadrant names instead of priority levels

## Technical Root Cause

**Data Model Mismatch:**
- JSON files store `eisenhower` field (values: `urgent-important`, `not-urgent-important`, etc.)
- Extension's `WorkItem` interface expects `priority` field (values: `critical`, `high`, `medium`, `low`)
- No mapping exists between Eisenhower quadrants and priority levels

**Impact Points:**
1. `itemLoader.ts`: Loads JSON directly without field mapping
2. `WorkItem` interface: Defines `priority: string` but source data has `eisenhower`
3. `getItemIconWithPriority()`: Expects Priority enum but receives undefined
4. Tooltip generation: Displays unmapped field value

## Expected Behavior

- Icons should be colored: Red (critical), Orange (high), Blue (medium), Gray (low)
- Tooltips should show: "Priority: critical/high/medium/low"
- Mapping: Q1 → critical, Q2 → high, Q3 → medium, Q4 → low

## Reproduction

1. Open any workspace with DevSteps items
2. View TreeView in any mode
3. Observe icon colors and hover tooltips

## Files Affected

- `packages/extension/src/treeView/types.ts` - WorkItem interface
- `packages/extension/src/treeView/utils/itemLoader.ts` - JSON loading
- `packages/extension/src/utils/icons.ts` - Icon color logic
- `packages/extension/src/treeView/nodes/workItemNode.ts` - Tooltip generation
