## Problem

`packages/extension/src/webview/renderers/traceabilityRenderer.ts:68` references `STATUS.DONE` inside a string blob that is injected as browser-side JavaScript. `STATUS` is never defined in browser scope — this causes a `ReferenceError` on every panel load, silently crashing the traceability graph before any GPU rendering code could run.

## Evidence

Found by `analyst-archaeology` (R1). Live production bug. Compound risk score: 60 (High). Referenced as QG-44 and S-02 in staleness registry. `STATUS` is a TypeScript enum exported from `packages/shared` — it is resolved at compile time in Node.js but never serialized for browser use.

## Fix

1. Resolve `STATUS` enum values before injection and serialize as JSON constants:
   ```typescript
   // Instead of injecting STATUS.DONE directly:
   const constants = JSON.stringify({ STATUS_DONE: ItemStatus.DONE });
   // Inject as: const CONSTANTS = ${constants};
   ```
2. Or: import status values in the SPA bundle (requires Phase 1 browser bundle work)

## Validation
- Open traceability panel in a project with items
- Verify no `ReferenceError` in webview DevTools console
- Run `npm run build && npm test`
