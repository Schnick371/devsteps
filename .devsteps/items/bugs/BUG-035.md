**Problem:**
Counter system used both lowercase ('story') and uppercase ('STORY') keys in counters.json, causing ID collisions.

**Symptoms:**
- Created STORY-001 instead of STORY-078
- counters.json contained both 'STORY': 77 and 'story': 1

**Root Cause:**
add.ts used args.type (lowercase) instead of uppercase prefix for counter keys

**Impact:**
Critical - prevents correct ID generation

**Fixed in commit:** 2297bea
