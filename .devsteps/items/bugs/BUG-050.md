# MCP Server shows wrong version (0.1.0 instead of package.json version)

## Problem

MCP Server version 0.8.4 was published to npm, but shows wrong version:

```bash
npx --package=@schnick371/devsteps-mcp-server@0.8.4 devsteps-mcp --version
# Shows: 0.1.0 (WRONG!)
# Expected: 0.8.4
```

## Root Cause

Hardcoded version strings in multiple files:
- `packages/mcp-server/src/index.ts` - `.version('0.1.0')`
- `packages/mcp-server/src/handlers/init.ts` - `version: '0.1.0'`
- `packages/mcp-server/src/logger.ts` - `version: '0.1.0'`
- `packages/mcp-server/src/metrics.ts` - `version: '0.1.0'`

## Impact

- ✅ Package published as 0.8.4 on npm
- ❌ Runtime shows 0.1.0 when executed
- ❌ Metrics report wrong version
- ❌ Logs show wrong version
- ❌ Confusing for users and debugging

## Solution

Same fix as TASK-186 (CLI already fixed):

```typescript
import packageJson from '../package.json' with { type: 'json' };

program.version(packageJson.version);
// And replace all hardcoded '0.1.0' with packageJson.version
```
