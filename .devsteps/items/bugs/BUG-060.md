## Problem
Three `FileSystemWatcher` instances in `packages/extension/src/extension.ts` (~L104, L160, L238) call `treeDataProvider.refresh()` with zero debounce. During DevSteps sprint runs (burst of `.result.json` writes via atomic `.tmp → rename`), this causes ~60 rapid-fire TreeView re-renders and visible UI freezes.

## Fix
- Add a shared `debounce(fn: () => void, ms: number): () => void` utility in `packages/extension/src/utils/`
- Wrap all three `refresh()` call sites with `debounce(refresh, 500)`
- Verify no race conditions exist (idempotent refresh is safe)

## Acceptance Criteria
- All 3 watcher callbacks debounced at 500ms
- Unit test: rapid-fire events → refresh called only once
