## Problem
`.devsteps/cbp/` and `.devsteps/analysis/` are not excluded by the generated `.gitignore` in `devsteps init`. MandateResult files, analysis envelopes, and sprint CBP data (which may include security analysis, file paths, API findings) can leak into version control in team repositories.

## Fix
- Extend the CLI `init` command's gitignore template to include `.devsteps/cbp/` and `.devsteps/analysis/`
- Patch `scripts/` or `packages/cli/src/commands/` where gitignore is generated
- Add migration note in README for existing projects

## Acceptance Criteria
- `devsteps init` generates `.gitignore` with `.devsteps/cbp/` and `.devsteps/analysis/` excluded
- Existing `.devsteps/.gitignore` (if present) also gets these entries
- Test: bats test verifies gitignore content after `devsteps init`
