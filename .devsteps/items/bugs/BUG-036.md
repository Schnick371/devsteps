# Problem

After STORY-079 items directory migration, hierarchical view shows **no items**, while flat view works correctly.

## Symptoms
- TreeView hierarchical mode: Empty (no Epics/Requirements displayed)
- TreeView flat mode: All items displayed correctly  
- No errors in developer console
- User reports: "im hirarchical view gibt es keine items, im gegensatz zum flat view"

## Root Cause
**File:** `packages/extension/src/treeView/nodes/hierarchyRootNode.ts` (Line 42-43)

```typescript
const indexPath = vscode.Uri.joinPath(workspaceRoot, '.devsteps', 'index.json');
const indexData = await vscode.workspace.fs.readFile(indexPath);
```

**Problem:** Code reads old monolithic index (`.devsteps/index.json`)

**Reality:** STORY-079 migrated to refs-style index structure:
- Old (removed): `.devsteps/index.json`
- New: `.devsteps/index/by-type/epics.json`, `requirements.json`, etc.

## Impact
- ❌ Hierarchical view completely unusable
- ❌ Blocks workflow for users preferring tree structure  
- ❌ Critical regression in 0.8.1-next.1 release
- ✅ Flat view unaffected (uses new index correctly)

## Expected Behavior
Hierarchical view should load top-level items from refs-style index:
- Scrum: Read `.devsteps/index/by-type/epics.json`
- Waterfall: Read `.devsteps/index/by-type/requirements.json`

## Reproduction
1. Install Extension 0.8.1 (pre-release)
2. Open project with `.devsteps/` directory
3. Switch TreeView to "Hierarchical" mode
4. Observe: No items displayed

## Related Items
- **STORY-079**: Items directory migration (introduced refs-style index)
- **BUG-002**: Previous hierarchical view fix (used old index structure)
