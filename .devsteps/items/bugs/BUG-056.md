## Bug Report

Since **v1.0.0** (MCP server), Markdown description files (`.devsteps/items/**/*.md`) are written with literal `\n` character sequences instead of real newlines.

## Symptoms

The `.md` file content looks like:
```
## Implemented\n\n### Show-ServerLayoutEditor — shared hostname warning\n- Build `nameToRoles` map before rendering table\n...
```

Instead of the expected:
```markdown
## Implemented

### Show-ServerLayoutEditor — shared hostname warning
- Build `nameToRoles` map before rendering table
...
```

## Root Cause

The `description` string arriving at `addItem()` / `updateItem()` contains literal two-character sequences `\n` (backslash + n) rather than real newline (`U+000A`) characters. This is how MCP protocol clients (e.g., GitHub Copilot v1.0.0+) may transmit multiline strings in certain contexts.

`writeFileSync` writes the raw string — no normalization occurs — so the file lands with literal `\n`.

## Affected Code

- `packages/shared/src/core/add.ts` — `description` written at line ~107
- `packages/shared/src/core/update.ts` — `description` / `append_description` written at lines ~127–134

## Fix

Add a `normalizeMarkdown(s: string): string` utility that replaces `\\n` → `\n` and `\\t` → `\t`, and apply it to `description` and `append_description` before every `writeFileSync`.

Candidate location for utility: `packages/shared/src/utils/text.ts` (new file) or inline in add/update.
