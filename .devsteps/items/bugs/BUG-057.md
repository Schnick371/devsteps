## Problem

`packages/extension/src/webview/utils/htmlHelpers.ts:27` uses `Math.floor(Math.random() * 62)` in a loop to generate the CSP nonce. `Math.random()` is not a CSPRNG — the nonce is statistically predictable, allowing an attacker with timing access to reconstruct the sequence and bypass CSP.

## Evidence

Found by `analyst-archaeology` (R1) and confirmed by `aspect-quality` (R2). Compound risk score: 72 (Critical). Referenced as QG-20 in quality gap registry.

## Fix

Replace with:
```typescript
export function getNonce(): string {
  return globalThis.crypto.getRandomValues(new Uint8Array(16))
    .reduce((acc, byte) => acc + byte.toString(16).padStart(2, '0'), '');
}
```

## Validation
- Replace nonce generation in `htmlHelpers.ts:27`
- Verify CSP nonces are 32-character hex strings
- Run `npm run build && npm test`
