## Problem
13 leaf node worker and aspect agents contain `"agent"` in their `tools[]` frontmatter array. Per Spider Web protocol:

> "Non-coord agents NEVER call runSubagent — all are Leaf Nodes"

The `"agent"` tool enables `runSubagent` invocation. Having it in leaf node tools[] creates a protocol drift risk: a capable model could invoke it and bypass the coord orchestration contract, causing uncontrolled agent chaining.

## Affected Agents (~13 files)
All of the following that have `"agent"` in `tools[]`:
- `worker-coder`, `worker-tester`, `worker-documenter`, `worker-devsteps`
- `worker-refactor`, `worker-integtest`, `worker-workspace`, `worker-guide-writer`
- `aspect-impact`, `aspect-constraints`, `aspect-quality`, `aspect-staleness`, `aspect-integration`
- `analyst-*` agents (check each)

## Fix
1. Audit all `.github/agents/devsteps-R[1-5]-*.agent.md` files
2. Remove `"agent"` from any `tools[]` array found
3. Do NOT remove from `devsteps-R0-coord*.agent.md` — coord legitimately uses this tool

## Audit Command
```bash
grep -l '"agent"' .github/agents/devsteps-R[1-5]-*.agent.md
```

## Acceptance Criteria
- No Ring 1–5 agent file contains `"agent"` in `tools[]`
- `grep -l '"agent"' .github/agents/devsteps-R[1-5]-*.agent.md` returns empty
- Ring 0 coord agents still have `"agent"` in tools[]
- Frontmatter CI task validates this per-ring
