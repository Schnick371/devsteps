## Problem
`devsteps-R1-analyst-quality.agent.md` contains an `agents:` field in its YAML frontmatter. Ring 1 analysts are **leaf nodes** — they must never dispatch subagents. Per the Spider Web protocol:

> "Non-coord agents NEVER call `runSubagent` — all are Leaf Nodes"

Having an `agents:` dispatch array in a Ring 1 analyst violates this invariant and creates drift risk: a sufficiently capable model could interpret the array and attempt to invoke a subagent, bypassing the coord orchestration contract.

## Root Cause
Likely a copy-paste from a coord agent template during initial authoring. The `agents:` array was not stripped when creating the analyst variant.

## Fix
1. Open `.github/agents/devsteps-R1-analyst-quality.agent.md`
2. Remove the `agents:` key and its value array from the YAML frontmatter
3. Verify no other Ring 1 analyst files contain `agents:` arrays
