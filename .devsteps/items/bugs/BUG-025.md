## Problem
Extension fails to load MCP server after installation due to missing npm dependencies (commander, etc.).

Error: `Cannot find package 'commander' imported from dist/mcp-server/index.js`

## Root Cause
- Extension bundles unbundled MCP server files (index.js requires node_modules)
- `vsce package --no-dependencies` excludes all node_modules
- MCP server crashes on startup

## Solution Architecture
Use existing PackageInstaller to install packages from npm:
1. Install @schnick371/devsteps-mcp-server from npm on activation
2. Install @schnick371/devsteps-cli from npm (optional but useful)
3. Install @schnick371/devsteps-shared from npm (dependency)
4. Remove bundled MCP server from extension package
5. Always use latest versions from npm

## Benefits
- No bundling complexity
- Users get latest published versions
- Simpler maintenance
- Proper dependency management
