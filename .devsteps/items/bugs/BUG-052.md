# Bug Description

When running `npx @schnick371/devsteps-mcp-server`, npm cannot determine which executable to run.

## Error Message
```
npm error could not determine executable to run
```

## Root Cause
Package name and binary name don't match:
- Package: `@schnick371/devsteps-mcp-server`
- Binary: `devsteps-mcp` (defined in package.json bin field)

npx without explicit binary name only works when package name = binary name.

## Impact
- Extension cannot use npx to auto-install MCP server
- Users cannot manually test with `npx @schnick371/devsteps-mcp-server`
- Breaks the "no version = @latest" strategy

## Current Workaround
Use explicit binary name:
```bash
npx -y @schnick371/devsteps-mcp-server devsteps-mcp
```

## Expected Behavior
`npx @schnick371/devsteps-mcp-server` should work without specifying binary name.

## Solution Options
1. Add default binary matching package name (preferred)
2. Update extension to specify binary name explicitly
3. Rename package to match binary name
